Reflections

Can you think of an analogy to describe HTML, CSS and the DOM?

How would you describe it to your non-technical friend?

HTML would seem like the skeleton, it provides the basic structure of the website. What order things go it and what kind of elements are used. CSS is the flesh on that skeleton. The same HTML can be made to look worlds apart using CSS just as most people have the same basic skeletal structure and yet look very different. The DOM is the HTML after the web browser has interpreted it. Often the web browser makes some adjustments and it is not exactly as you wrote it - side note - I am still way away from understanding DOM.

What is meant by boxifying design?

Webpages are designed in a grid-like manner. Boxifying design is where one breaks up the visuals of the webpage into square/rectangular segments, from the largest square into the smallest squares. Example below:

What is the box model?

Every element in web design is a rectangular box. This box is constructed with different layers as shown below. The inner layer is of course, the content; this could be text, an image or simply empty space. We then have padding which the web developer can add to acheive empty space around the content that lies within the border. We then have the border, and outside of this the margin. The margin is empty space held outside of the element box which is useful to give room between elements so the web page doesn't get too crowded.
